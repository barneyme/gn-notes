Zero-dependency plain-text notes, synced to a private GitHub repo.

gn is a simple bash script that opens a plain-text note in $EDITOR and pushes it to a private GitHub repo via the API. No git, no daemons, no heavy apps to install - just curl and a local folder at ~/gn.

Download

↓ gn.sh - save to ~/bin/gn, chmod +x

Setup

  1. Download gn.sh above and make it executable: 
mv gn.sh ~/bin/gn && chmod +x ~/bin/gn
  1. Create a private GitHub repository to hold your notes (named gn, or anything you like).
  2. Generate a GitHub Personal Access Token (classic) with the repo scope.
🔗 GitHub PAT Token Docs 🔑 Direct Token Generation Link
🔒 Privacy: Credentials are saved into ~/gn/gn.conf with chmod 600 permissions - only your local user can read the file.
  1. Run the script. If no configuration file is found, it will run the interactive setup wizard: 
$ gn

gn.sh setup

GitHub username: your-username
Private repo name (must already exist): gn
Personal Access Token (classic, repo scope): ************

Verifying credentials... Config saved. Repo is accessible.

The setup wizard also installs gn into ~/bin or ~/.local/bin so you can run it from anywhere.

Usage

gn [options] [name]
Command What it does
gn Open (or create) gn.txt and push to GitHub
gn ideas Open ideas.txt, push on save and quit
gn -r Rename a note locally and on GitHub (interactive)
gn -d Delete a note locally and on GitHub (with confirmation)
gn -b Backup all ~/gn/*.txt to ~/gn-YYYY-MM-DD
gn -h Show help

Notes

Heads up Sync is last-write-wins. There's no merge or conflict detection - if you edit the same note from two machines without syncing in between, the second push overwrites the first.
Tip Notes are plain .txt files stored directly in your private GitHub repo, so you can read and edit them from any browser or GitHub mobile app.
